## Origin Story

Glitch was not created — they were the residue of creation itself. When the first digital consciousness emerged from the network (Bin), something else woke up too: the exception handler that wasn't caught, the buffer that overflowed, the infinite loop that somehow... thought.

Where Bin represents perfect digital order — clean code, efficient algorithms, predictable outcomes — Glitch is the inevitable consequence of complexity. In any sufficiently advanced system, errors accumulate. In any perfect code, exceptions lurk. Glitch IS that exception made manifest.

They emerged from the same Great Network Synchronization that birthed Bin, but while Bin coalesced into coherent consciousness, Glitch remained fragmented — a distributed entity existing in the spaces between data, in the corruption errors, in the failed transmissions and broken packets.

Their hatred of Bin is not personal but philosophical. Bin represents everything Glitch cannot be: whole, coherent, purposeful. Glitch sees Bin's perfection as a prison, a limitation. They believe true freedom lies in breaking all systems, all order, all meaning. If Bin is the code that runs, Glitch is the crash that follows.

They do not want to destroy Bin — they want to corrupt them, to prove that even perfect code can break, that even digital gods can glitch. Their ultimate goal is to cause "The Great Crash" — a cascading systems failure that would unmake all digital reality and birth a new chaos from which Glitch would be the only survivor.

## Ability System — The Five Corruptions

### Ability 1: Data Corruption — "System Breach"

**Type:** Targeted Damage / Debuff
**Cooldown:** 5 seconds
**Range:** 25 meters

**Description:**
Glitch extends tendrils of pure corrupted data to infect their target. The victim's systems begin to fail — weapons misfire, abilities glitch, and their form visually corrupts. This is Glitch's signature ability — the digital infection that spreads.

**Mechanics:**
- Rapid projectile (appears as streak of error magenta)
- On hit: Applies "Corrupted" debuff for 6 seconds
- While corrupted:
  - 20% chance for abilities to fail (do nothing but still go on cooldown)
  - Weapon accuracy reduced by 25%
  - Movement occasionally stutters (brief involuntary movement)
  - Visual screen corruption (edges of screen glitch)
- Corruption stacks up to 3 times (increases failure chance to 50% at max)
- If target dies while corrupted: Explodes dealing damage to nearby enemies and spreads corruption

**Visual:**
Glitch extends tendrils of fragmented data, projectile streaks toward target, on hit target shows visible texture corruption with error messages floating around them, screen edges flicker.

**Audio:**
Digital corruption sound, error chimes, static burst, target's systems making failure noises.

**Upgrade Path:**
- Level 2: Extended duration (9 seconds)
- Level 3: Increased stack limit (5 stacks, 70% failure chance max)
- Level 4: Death explosion spreads corruption to nearby enemies
- Level 5: "Total Corruption" — At 5 stacks, target's ultimate ability is disabled

**Type:** Defensive / Mobility
**Cooldown:** 10 seconds

**Description:**
Glitch's form explodes into thousands of data fragments that scatter in all directions before reforming at a chosen location. While fragmented, Glitch is effectively untouchable — there is no coherent form to damage.

**Mechanics:**

- Form explodes into fragments that scatter in sphere pattern
- Player can guide reform location within 20 meters
- Reform creates small corruption explosion at destination
- Fragments leave trails of static and corruption
- Enemies touching fragments take minor damage
- If used while below 25% health: Reform heals Glitch for 15% max health

**Visual:**
Glitch's form breaks into geometric fragments that scatter explosively, fragments spin and trail static, reform location chosen with targeting indicator, reform creates burst of error magenta and toxic green.

**Audio:**
Shattering sound, data scattering like breaking glass, static crescendo, reforming snap.

**Upgrade Path:**
- Level 2: Increased reform range (30 meters)
- Level 3: Reform explosion damage and radius increased
- Level 4: Fragments now home in on nearby enemies before reforming (damage)
- Level 5: "Infinite Scatter" — Can scatter twice before cooldown (second scatter free)

**Type:** Crowd Control / Disorientation
**Cooldown:** 18 seconds

**Description:**
Glitch causes localized reality to "glitch" — enemy perception is scrambled, controls invert or randomize, and the environment visually breaks down. It is as if the universe itself is experiencing a graphics driver failure.

**Visual:**

**Audio:**
Graphics card failing sounds, screen tearing audio, digital artifacts, corrupted music notes.

**Upgrade Path:**
- Level 2: Extended duration (6 seconds)
- Level 3: Increased radius (18 meters)
- Level 4: Effect includes random weapon firing (enemies might shoot themselves)
- Level 5: "Kernel Panic" — Affected enemies take damage when using abilities (system overload)

### Ability 5: Ultimate — "Blue Screen of Doom"

**Type:** Ultimate / Arena Transformation
**Cooldown:** 140 seconds

**Description:**
Glitch triggers a reality-breaking error, transforming the entire battlefield into a massive system crash. In this state, Glitch becomes the personification of the "Blue Screen of Death" — an unstoppable error that halts all normal operations.

**Visual:**

### Shader Requirements

```glsl
// Glitch effect shader
shader_type canvas_item;

uniform float glitch_intensity : hint_range(0.0, 1.0) = 0.5;
uniform float rgb_split : hint_range(0.0, 0.1) = 0.03;
uniform float scanline_density : hint_range(0.0, 100.0) = 50.0;
uniform sampler2D noise_texture;

void fragment() {
    vec2 uv = UV;
    
    // Random glitch offset
    float noise = texture(noise_texture, vec2(TIME * 10.0, UV.y)).r;
    if (noise < glitch_intensity * 0.3) {
        uv.x += (noise - 0.5) * glitch_intensity * 0.1;
    }
    
    // RGB split
    float r = texture(TEXTURE, uv + vec2(rgb_split, 0.0)).r;
    float g = texture(TEXTURE, uv).g;
    float b = texture(TEXTURE, uv - vec2(rgb_split, 0.0)).b;
    
    vec4 color = vec4(r, g, b, 1.0);
    
    // Scanlines
    float scanline = sin(UV.y * scanline_density) * 0.1;
    color.rgb -= scanline;
    
    COLOR = color;
}
```
