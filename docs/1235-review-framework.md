# 1/2/3/5 Master Review Framework
## Operational Review Methodology v1.0

### Framework Overview

The 1/2/3/5 Review System is a rigorous, multi-phase quality assurance methodology designed for complex agent-based operations. It ensures thorough analysis, iterative improvement, and comprehensive documentation.

**Structure:**
- **1** Review Report
- **2** Success Deliverables Required for Completion
- **3** Review Report Recommendations (each with 1 paragraph + 1 parent bullet + 5 sub-bullets)
- **5** Sub-bullet composition (1 addition, 1 update, 1 removal, 2 flex)

**Phases:**
1. Review 1 → Implement recommendations
2. Review 2 (incorporating Review 1 changes) → Implement recommendations  
3. Review 3 (incorporating all previous changes) → Implement recommendations
4. Master Critical Review (unification of 3 reviews + +1 edits to 9 layers)
5. Triple Pass Final Verification (read-only → edits → verification)
6. Fine-tuning with sub-agent optimization
7. Operational Plan Documentation
8. Archive Consolidation

## Appendix: Usage Instructions

### When to Use 1/2/3/5 Review

**Appropriate For:**
- Complex multi-component projects
- Architecture decisions with long-term implications
- Quality-critical deliverables
- Cross-functional initiatives

**Not Recommended For:**
- Simple bug fixes
- Routine maintenance
- Emergency hotfixes

### Initiating a Review

1. **Define Scope:** What exactly is being reviewed?
2. **Set Boundaries:** What is in-scope vs out-of-scope?
3. **Allocate Time:** Each review phase requires significant time
4. **Prepare Resources:** Ensure access to all necessary tools/information

### Review Checklist

**Before Starting:**
- [ ] Scope clearly defined
- [ ] Stakeholders identified
- [ ] Resources available
- [ ] Timeline established

**During Review:**
- [ ] Following 1/2/3/5 structure strictly
- [ ] Documenting all findings
- [ ] Not skipping phases

**After Review:**
- [ ] All recommendations implemented
- [ ] Archive created
- [ ] Lessons learned captured
