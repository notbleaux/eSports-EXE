# RPI 执行手册

## 适用范围

将 RPI 作为通用工程执行法，适用于：

- 跨模块改动
- 需求不完整或约束不清晰
- 高风险重构与修复
- 需要多轮迭代优化的任务

## 阶段检查清单

### 调研

- 明确问题定义和验收标准。
- 梳理相关代码、依赖、配置与数据边界。
- 记录事实、假设、未知项。
- 提炼进入计划阶段所需的关键结论。

### 计划

- 将目标拆为可执行步骤。
- 标注每步影响范围与文件。
- 定义风险、缓解与回滚策略。
- 定义验证命令和成功标准。

### 实施

- 严格按已确认步骤执行。
- 小步提交改动并持续验证。
- 出现阻塞时暴露问题并更新计划。
- 控制范围，避免无关扩散。

### 迭代

- 复核功能正确性和失败路径。
- 检查边界条件、性能与安全。
- 识别测试缺口和维护成本问题。
- 形成优先级清晰的下一轮行动项。

## 通用提示词模板

### 调研模板

```text
进入调研阶段，不要改代码。
目标：<目标>。
输出：概要 / 关键发现 / 上下文 / 待确认问题 / 建议。
```

### 计划模板

```text
进入计划阶段，不要改代码。
基于已确认事实，输出：目标 / 前提假设 / 范围与涉及文件 / 执行步骤 / 风险与缓解 / 验证方案。
```

### 实施模板

```text
进入实施阶段，按已确认计划执行并持续验证。
每次更新输出：已完成改动 / 验证结果 / 偏离说明 / 剩余工作。
```

### 迭代模板

```text
进入迭代阶段，对当前实现做批判性复盘。
输出：优点 / 问题 / 边界情况与缺口 / 优先级改进项。
```

## 回退策略

- 发现关键前提错误：回到调研阶段重新确认事实。
- 实施中出现计划外风险：暂停并回到计划阶段更新方案。
- 复盘发现高优先级缺陷：开启下一轮“计划 -> 实施”。
